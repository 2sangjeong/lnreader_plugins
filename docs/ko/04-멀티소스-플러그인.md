# 멀티소스 플러그인 (Multisrc)

같은 CMS(워드프레스 테마 등)를 쓰는 사이트 여러 개를 **템플릿 하나로 자동 생성**하는 시스템입니다.

---

## 왜 멀티소스인가?

예를 들어 Madara 워드프레스 테마를 사용하는 사이트가 50개 있다면, 각각 플러그인을 만들면 거의 동일한 코드를 50번 작성해야 합니다. 멀티소스는 이 문제를 해결합니다.

```
plugins/multisrc/madara/
├── template.ts       ← 공통 로직 (HTML 파싱, API 호출 등)
├── sources.json      ← 사이트 목록 (URL, ID, 이름 등)
├── generator.js      ← 자동 생성 스크립트
└── filters/
    ├── site1.json    ← site1 전용 필터
    └── site2.json    ← site2 전용 필터
```

---

## 폴더 구조 상세

### `template.ts` - 공통 로직

모든 해당 사이트에서 공통으로 사용하는 파싱 로직이 들어있습니다.

```typescript
// template.ts 예시 구조
export class MadaraPlugin implements Plugin.PluginBase {
  constructor(source) {
    this.id = source.id;
    this.name = source.sourceName;
    this.site = source.url;
    // ...
  }

  async popularNovels(pageNo, { showLatestNovels, filters }) {
    // Madara 테마의 공통 인기작 가져오기 로직
  }

  async parseNovel(novelPath) {
    // Madara 테마의 공통 소설 파싱 로직
  }
  // ...
}
```

### `sources.json` - 사이트 목록

```json
[
  {
    "id": "site1",
    "sourceName": "Site One",
    "url": "https://site1.com",
    "options": {
      "lang": "English"
    }
  },
  {
    "id": "site2",
    "sourceName": "Site Two",
    "url": "https://site2.com",
    "options": {
      "lang": "Korean",
      "customJs": "// 이 사이트만 특별히 처리할 JS 코드"
    }
  }
]
```

### `generator.js` - 자동 생성 스크립트

`sources.json`의 각 항목을 읽어 `template.ts`와 합쳐서 플러그인 파일을 생성합니다.

```javascript
// generator.js가 하는 일
sources.forEach(source => {
  // template.ts 읽기
  // source 정보를 template에 주입
  // plugins/<lang>/<Name>multisrc.ts 파일 생성
});
```

생성 결과물 예시:
```
plugins/english/SiteOnemultisrc.ts   ← 자동 생성된 파일
plugins/korean/SiteTwomultisrc.ts    ← 자동 생성된 파일
```

## 새 사이트를 멀티소스에 추가하는 방법

### 1. 맞는 템플릿 찾기

새 사이트가 어떤 CMS를 쓰는지 확인합니다. 사이트 소스에서 테마 이름을 찾거나, 기존 플러그인과 HTML 구조를 비교합니다.

### 2. `sources.json`에 항목 추가

```json
// plugins/multisrc/madara/sources.json
[
  // ... 기존 항목들 ...
  {
    "id": "newsite",               // 고유 ID (소문자, 영문)
    "sourceName": "New Site",      // 표시 이름
    "url": "https://newsite.com",  // 사이트 URL
    "options": {
      "lang": "English"            // 소설 언어
    }
  }
]
```

### 3. 아이콘 추가

```
icons/src/eng/newsite/icon.png   (96x96px)
```

### 4. 플러그인 재생성

```bash
npm run build:multisrc
```

### 5. 필터가 필요하면 추가 (선택)

```
plugins/multisrc/madara/filters/newsite.json
```

```json
{
  "filters": {
    "genres": {
      "type": "CheckboxGroup",
      "label": "Genres",
      "value": [],
      "options": [
        { "label": "Action", "value": "action" },
        { "label": "Romance", "value": "romance" }
      ]
    }
  }
}
```

## 새 멀티소스 템플릿을 만드는 방법

완전히 새로운 CMS를 지원하려면 새 템플릿을 만듭니다.

### 1. 폴더 생성

```
plugins/multisrc/mycms/
├── template.ts
├── sources.json
└── generator.js
```

### 2. `template.ts` 작성

```typescript
// 파라미터를 생성자로 받는 클래스 작성
export class MyCMSPlugin implements Plugin.PluginBase {
  id: string;
  name: string;
  icon: string;
  site: string;
  version: string;

  constructor(source: any) {
    this.id = source.id;
    this.name = source.sourceName;
    this.icon = `src/${source.options?.lang?.toLowerCase() || 'eng'}/${source.id}/icon.png`;
    this.site = source.url;
    this.version = '1.0.0';
  }

  async popularNovels(pageNo, options) {
    // 공통 파싱 로직 작성
  }

  // ... 나머지 메서드들
}
```

### 3. `generator.js` 작성

기존 generator.js를 복사해서 파일명만 변경합니다:

```javascript
import list from './sources.json' with { type: 'json' };
import { readFileSync } from 'fs';
import { fileURLToPath } from 'url';
import { dirname, join } from 'path';

const folder = dirname(fileURLToPath(import.meta.url));

export const generateAll = function () {
  return list.map(source => {
    return generator(source);
  });
};

const generator = function (source) {
  const template = readFileSync(join(folder, 'template.ts'), { encoding: 'utf-8' });
  const pluginScript = `
${template}
const plugin = new MyCMSPlugin(${JSON.stringify(source)});
export default plugin;
  `.trim();
  return {
    lang: source.options?.lang || 'English',
    filename: source.sourceName.replace(/[^\w]/g, ''),
    pluginScript,
  };
};
```

### 4. `sources.json` 작성

```json
[
  {
    "id": "firstsite",
    "sourceName": "First Site",
    "url": "https://firstsite.com",
    "options": { "lang": "English" }
  }
]
```

### 5. 메인 생성 스크립트에 등록

`plugins/multisrc/generate-multisrc-plugins.js`를 열어서 새 템플릿을 추가합니다.

---

## 자동 생성 과정 확인

```bash
npm run build:multisrc
```

실행하면 콘솔에 이런 출력이 나옵니다:
```
[madara] Generating: site1               🔎with filters🔍
[madara] Generating: site2               🚫no filters🚫
```

생성된 파일은 `plugins/<언어>/`에서 확인할 수 있습니다.

---

## 주의사항

- `*multisrc.ts`로 끝나는 파일은 자동 생성된 파일입니다 - **직접 수정하지 마세요**
- 수정이 필요하면 `template.ts`나 `sources.json`을 수정하고 재생성하세요
- git에서 자동 생성 파일을 커밋할 때는 `build:multisrc`를 먼저 실행하세요
