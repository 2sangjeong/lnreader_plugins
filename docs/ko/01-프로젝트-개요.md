# LNReader 플러그인 프로젝트 개요

이 문서는 `lnreader-plugins` 저장소의 전체 구조와 동작 방식을 설명합니다.

---

## 이 프로젝트가 하는 일

[LNReader](https://github.com/LNReader/lnreader)는 안드로이드 라이트노벨 리더 앱입니다.
이 저장소는 앱이 **다양한 소설 사이트에서 글을 가져올 수 있도록** 플러그인을 모아두는 곳입니다.

플러그인은 각 소설 사이트마다 하나씩 만들며, 앱은 플러그인을 통해:
- 인기 소설 목록을 가져오고
- 소설 정보(제목, 작가, 장르 등)를 가져오고
- 챕터 내용을 가져옵니다

---

## 폴더 구조

```
lnreader-plugins/
│
├── plugins/                 ← 실제 플러그인 파일들이 있는 핵심 폴더
│   ├── english/             ← 영어 소설 사이트 플러그인
│   ├── korean/              ← 한국어 소설 사이트 플러그인 (북토끼 등)
│   ├── chinese/             ← 중국어
│   ├── japanese/            ← 일본어
│   ├── ... (14개 언어)
│   ├── multi/               ← 여러 언어를 지원하는 사이트
│   ├── multisrc/            ← 템플릿으로 자동 생성되는 플러그인 모음
│   └── index.ts             ← 모든 플러그인을 모아주는 자동 생성 파일
│
├── src/                     ← 웹 테스트 인터페이스 (React 앱)
│   ├── lib/                 ← fetch, storage 등 핵심 유틸리티
│   ├── types/               ← TypeScript 타입 정의
│   └── libs/                ← 플러그인에서 @libs/... 로 import하는 라이브러리
│
├── icons/src/               ← 플러그인 아이콘 파일 (96x96px)
├── public/static/           ← 빌드된 아이콘이 복사되는 곳
├── scripts/                 ← 빌드 자동화 스크립트
└── docs/                    ← 문서 폴더 (현재 위치)
```

---

## 플러그인의 두 가지 종류

### 1. 단일 플러그인 (Single Plugin)
- `plugins/english/nobleMTL.ts` 처럼 하나의 `.ts` 파일이 하나의 사이트를 담당
- 사이트마다 HTML 구조가 달라서 직접 파싱 로직을 작성

### 2. 멀티소스 플러그인 (Multisrc Plugin)
- 같은 CMS(워드프레스 테마 등)를 쓰는 사이트가 많아서 **템플릿 하나로 여러 플러그인을 자동 생성**
- `plugins/multisrc/madara/` 폴더가 대표적인 예
- `sources.json`에 사이트 목록을 추가하면 플러그인이 자동 생성됨

## 빌드 파이프라인 (코드가 앱에 전달되기까지)

```
1. 멀티소스 생성
   plugins/multisrc/*/generator.js 실행
   → plugins/<언어>/<이름>multisrc.ts 파일 생성

2. 플러그인 인덱스 생성
   scripts/generate-plugin-index.js 실행
   → plugins/index.ts 생성 (모든 플러그인 import/export)

3. TypeScript 컴파일
   tsc --project tsconfig.production.json
   → .dist/ 폴더에 JS 파일 생성

4. 매니페스트 생성
   scripts/build-plugin-manifest.js 실행
   → .dist/plugins.json 생성 (앱이 다운로드하는 플러그인 목록)

5. 배포 (CI/CD)
   master 브랜치에 push → GitHub Actions가 plugins 브랜치에 자동 배포
```

## 개발 시작하기

```bash
# 의존성 설치
npm install

# 개발 서버 실행 (localhost:3000에서 플러그인 테스트 가능)
npm run dev:start

# 전체 빌드
npm run build:full

# 코드 스타일 검사
npm run lint
npm run format:check
```
