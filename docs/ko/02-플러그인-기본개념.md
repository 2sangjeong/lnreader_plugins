# 플러그인 기본 개념

플러그인이 어떻게 동작하는지 핵심 구조를 설명합니다.

---

## 플러그인은 클래스다

모든 플러그인은 `Plugin.PluginBase` 인터페이스를 구현하는 TypeScript 클래스입니다.

```typescript
import { Plugin } from '@/types/plugin';

class MyPlugin implements Plugin.PluginBase {
  // 필수 속성들
  id = 'my-plugin';
  name = '내 플러그인';
  icon = 'src/eng/myplugin/icon.png';
  site = 'https://example.com';
  version = '1.0.0';

  // 필수 메서드들 (아래에서 설명)
  async popularNovels(...) { ... }
  async parseNovel(...) { ... }
  async parseChapter(...) { ... }
  async searchNovels(...) { ... }
}

// 파일 마지막에 반드시 인스턴스를 export해야 함
export default new MyPlugin();
```

---

## 필수 속성 설명

| 속성 | 타입 | 설명 |
|------|------|------|
| `id` | `string` | 플러그인의 고유 식별자. 한 번 정하면 바꾸지 말 것 |
| `name` | `string` | 앱에 표시되는 플러그인 이름 |
| `icon` | `string` | 아이콘 파일 경로 (`public/static/` 기준) |
| `site` | `string` | 소설 사이트 URL |
| `version` | `string` | 플러그인 버전 (semver 형식: `1.0.0`) |

### 버전 규칙

```
1 . 0 . 0
↑   ↑   ↑
│   │   └─ patch: 셀렉터 수정 등 작은 버그 픽스
│   └───── minor: 필터 추가/제거 등 기능 개선
└───────── major: 사이트 URL 변경 등 큰 변화
```

---

## 필수 메서드 4가지

### 1. `popularNovels` - 인기 소설 목록

앱에서 메인 화면을 열 때 호출됩니다.

```typescript
async popularNovels(
  pageNo: number,                                          // 페이지 번호 (1부터 시작)
  { showLatestNovels, filters }: Plugin.PopularNovelsOptions<typeof this.filters>
): Promise<Plugin.NovelItem[]>
```

- `pageNo`: 몇 번째 페이지인지 (무한스크롤처럼 1, 2, 3... 증가)
- `showLatestNovels`: `true`면 최신 소설 정렬, `false`면 인기순
- `filters`: 사용자가 설정한 필터 값들
- **반환**: `NovelItem` 배열

```typescript
// NovelItem 구조
{
  name: '소설 제목',          // 필수
  path: '/novel/123',        // 필수 - 사이트 내 경로 (전체 URL이 아님!)
  cover: 'https://...',      // 선택 - 표지 이미지 URL
}
```

> **주의**: `path`는 전체 URL이 아니라 사이트 내 경로입니다.
> 예: `https://example.com/novel/123` → path는 `/novel/123`

---

### 2. `parseNovel` - 소설 상세 정보 + 챕터 목록

특정 소설 페이지를 열 때 호출됩니다.

```typescript
async parseNovel(novelPath: string): Promise<Plugin.SourceNovel>
```

- `novelPath`: `popularNovels`에서 반환한 `NovelItem.path` 값

```typescript
// SourceNovel 구조
{
  path: novelPath,           // 필수 - 입력받은 path 그대로
  name: '소설 제목',          // 필수
  cover: 'https://...',      // 선택
  author: '작가명',           // 선택
  artist: '삽화가',           // 선택
  genres: '판타지,로맨스',     // 선택 - 쉼표로 구분
  status: NovelStatus.Ongoing, // 선택 - 연재 상태
  summary: '줄거리...',       // 선택
  rating: 4.5,               // 선택 - 5점 만점
  chapters: [                // 챕터 목록
    {
      name: '1화',            // 필수
      path: '/chapter/1',    // 필수
      chapterNumber: 1,      // 선택 - 챕터 번호
      releaseTime: '2024-01-01', // 선택 - "YYYY-MM-DD" 형식
    }
  ]
}
```

---

### 3. `parseChapter` - 챕터 내용

챕터를 읽을 때 호출됩니다.

```typescript
async parseChapter(chapterPath: string): Promise<string>
```

- `chapterPath`: `ChapterItem.path` 값
- **반환**: 챕터의 HTML 문자열

```typescript
// 예시
async parseChapter(chapterPath: string): Promise<string> {
  const response = await fetchApi(this.site + chapterPath);
  const body = await response.text();
  const $ = loadCheerio(body);

  // 광고, 불필요한 태그 제거
  $('script, .ad-banner').remove();

  // 본문 HTML 반환
  return $('#chapter-content').html() || '';
}
```

---

### 4. `searchNovels` - 검색

검색창에서 검색할 때 호출됩니다.

```typescript
async searchNovels(
  searchTerm: string,  // 사용자가 입력한 검색어
  pageNo: number       // 페이지 번호
): Promise<Plugin.NovelItem[]>
```

- **반환**: `NovelItem` 배열 (popularNovels와 동일한 구조)

## 선택 속성들

### `filters` - 검색 필터

인기 소설 화면에 필터 버튼을 추가합니다. 자세한 내용은 [필터 시스템 문서](./06-필터-시스템.md) 참고.

### `imageRequestInit` - 이미지 요청 설정

사이트가 이미지 핫링킹을 막을 때 사용합니다.

```typescript
imageRequestInit: Plugin.ImageRequestInit = {
  headers: {
    Referer: 'https://example.com',  // 이걸 추가하면 이미지가 로드됨
  },
};
```

### `resolveUrl` - 전체 URL 변환

앱이 path를 전체 URL로 변환해야 할 때 사용합니다.

```typescript
resolveUrl = (path: string, isNovel?: boolean) =>
  this.site + path;
```

### `webStorageUtilized` - 웹 스토리지 사용 여부

플러그인이 `localStorage`나 `sessionStorage`를 사용할 때 `true`로 설정합니다.

```typescript
webStorageUtilized = true;
```
