# 웹 테스트 인터페이스

`npm run dev:start`로 실행하는 로컬 웹 서버에서 플러그인을 쉽게 테스트할 수 있습니다.

---

## 실행 방법

```bash
npm run dev:start
# http://localhost:3000 에서 접속
```

---

## 화면 구성

### 왼쪽 사이드바 - 플러그인 선택

- 모든 플러그인 목록이 표시됩니다
- 검색창으로 플러그인을 필터링할 수 있습니다
- 클릭하면 해당 플러그인 테스트 화면으로 이동합니다

### 오른쪽 메인 영역 - 탭으로 구분된 테스트 화면

#### Popular Novels 탭

`popularNovels()` 함수를 테스트합니다.

- **Latest** 버튼: `showLatestNovels = true`로 호출
- **필터 UI**: 플러그인에 `filters`가 있으면 필터 버튼 표시
- 페이지 번호를 입력하고 호출 가능

#### Search Novels 탭

`searchNovels()` 함수를 테스트합니다.

- 검색어를 입력하고 검색
- 결과 목록 확인

#### Parse Novel 탭

`parseNovel()` 함수를 테스트합니다.

- URL Path를 직접 입력하거나 목록에서 소설을 클릭하면 자동 입력
- 소설 상세 정보와 챕터 목록 확인

#### Parse Chapter 탭

`parseChapter()` 함수를 테스트합니다.

- Chapter Path를 직접 입력하거나 챕터 목록에서 클릭하면 자동 입력
- 챕터 HTML 내용 확인

#### Settings 탭

플러그인에 `pluginSettings`가 있으면 설정 UI가 표시됩니다.

---

## CORS 프록시

일반적으로 브라우저에서 다른 도메인으로 요청하면 CORS 오류가 납니다.
개발 서버는 Vite의 프록시 미들웨어를 통해 이를 우회합니다.

`fetchApi`를 사용하면 자동으로 프록시를 거칩니다.

---

## 주의사항

- `localStorage`, `sessionStorage`는 웹 테스트에서 동작하지 않습니다 (앱 전용)
- 일부 사이트는 봇 차단이 강해서 테스트 서버에서도 접근이 안 될 수 있습니다
- 플러그인 수정 후 테스트하려면 새로고침하면 됩니다 (Vite HMR 지원)

---

## 디버깅 팁

브라우저 개발자 도구(F12) → Console에서:
- `fetchApi` 호출 시 URL과 헤더가 출력됩니다
- 에러 메시지 확인 가능

Network 탭에서:
- 실제 요청/응답 확인
- 응답 HTML 내용 검사
